Service

기업들이 능력 있는 개발자를 적극적으로 찾는 상황에서 기존의 채용 프로세스로는 기업이 요구하는 개발자를 찾는 데 한계가 있었습니다. 이러한 문제를 해결하기 위해 프로그래머스는 역량 있는 개발자들을 모아서 좋은 기업들과 연결할 수 있는 개발자 채용에 특화된 플랫폼을 출시했습니다.

Typography

Colors

Career Design

프로그래머스 채용서비스는 기술 중심의 이력서를 바탕으로 개발자 평가를 통해 취업활동을 진행할 수 있는 개발자 전문 서비스입니다.

1. 메인 페이지 2. 채용 프로그램 3. 채용 공고

커리어 메인페이지는 채용 서비스와 관련된 모든 컨텐츠와 기능이 담겨있는 시작점으로, 트래픽이 가장 많은 페이지입니다.

이 페이지는 서비스의 핵심이므로, 개발자들의 성공적인 구직활동을 위해 팀원들과 함께 지속적으로 서비스를 개선했습니다.

채용 프로그램

프로그래머스 채용 프로그램은 단 한 번의 코딩 테스트와 한 장의 이력서로 다양한 기업에 동시에 지원할 수 있어서, 구직 활동을 하는 개발자들의 참여율이 매우 높은 프로그램입니다.

Problem

프로그램에 참여하는 개발자 입장에서 몇가지 어려움이 있어서 지원 마감일까지 불필요한 눈치게임을 해야만 했습니다.

  • 지원 완료 이후 선택한 회사를 바꿀 수 없어서 선택이 어려움
  • 과거 지원 히스토리를 보고 싶은데 조회 불가
  • 테스트를 다시 보면 이전 점수보다 오히려 떨어질까봐 걱정
  • 지원하고 싶은 회사에 다른 지원자들이 얼마나 지원 했는지 궁금
Solution

팀원들과 함께 개발자 채용 과정에서 개발자들이 겪는 불필요한 부담을 줄여주기 위해 아래와 같이 서비스를 개선했습니다.

  • 지원 완료 이후 내가 선택한 회사를 변경 할 수 있는 기능 추가
  • 과거 지원 히스토리, 결과, 상태를 볼 수 있는 페이지 생성
  • 개발 실력이 검증된 상위 개발자 대상으로 테스트 과정 면제
  • 회사 로고, 지원율 정보를 추가해서 관련 정보 파악에 도움 되도록 개선
1. 선택한 회사 변경 기능

프로그램 지원 완료 이후 선택한 회사를 변경할 수 있는 기능을 추가했습니다. 이전에는 해당 기능이 없어서 수정을 원할 경우 지원을 취소하고 처음부터 다시 진행해야 하는 불편함이 있었습니다.

2. 프로그램 지원 내역

서비스 초기에는 채용 프로그램에 참여한 개발자의 지원 내역 진행 상태를 조회할 수 있는 기능이 부재하여, 이전 지원 내역에 대한 문의와 함께 해당 기능을 개발해달라는 많은 VOC가 접수되었습니다.

사용자들의 피드백을 적극 반영하여 해당 기능을 개발하였고, 그 결과 운영팀에 접수되는 관련 VOC가 크게 감소하였으며 사용자 만족도 역시 크게 향상되었음을 확인할 수 있었습니다.

3. 프리패스 기능

양질의 지원자 참여율을 높이기 위해, 이전 프로그램 테스트에 참여하여 개발 실력이 검증된 상위 개발자를 대상으로 테스트 과정을 면제해주는 '프리패스' 기능을 개발했습니다.

4. 프로그램 정보 페이지 개선

서비스 초기에는 참여 기업 수가 적어 채용 공고 목록 페이지를 텍스트로만 표시해도 사용자들이 읽기에 큰 어려움이 없었습니다. 그러나 참여 기업 수가 40개 이상으로 증가하면서 텍스트로만 구성된 테이블 정보가 잘 읽히지 않는 문제가 발생했습니다. 이에 따라 로고를 추가하여 테이블 정보를 정리하고, 지원율을 표시하여 개발자들이 회사 선택하는데 도움이 되도록 페이지 디자인을 개선했습니다.

채용 공고

채용 공고 페이지는 지원자들이 해당 포지션에 지원하기 위해 필수로 거쳐야 하는 채용 프로세스의 시작점입니다. 이 페이지는 지원자의 입사 결정에 큰 영향을 미치며, 텍스트 정보가 많이 포함된 중요한 페이지입니다. 따라서, 지원자들이 빠르게 정보를 스캔하고 쉽게 이해할 수 있도록 디자인을 구성하였습니다.

Problem

개발자들이 채용 포지션 목록과 공고 페이지만 보고 나한테 맞는 포지션을 찾는데 어려움을 겪고 있었습니다.

  • 내가 관심있을만한 회사를 추천받거나 모아서 보고 싶음
  • 회사 페이지 마다 정보의 양이 너무 달라서 어떤 회사는 읽기 어려움
  • 채용 지원을 하려고 이력서를 선택하고 지원하는 과정이 심리적으로 부담스러움
  • 내가 지원한 포지션 관리 페이지가 너무 불친절함 뭘 어떻게 해야될지 모르겠음
Solution

서비스 내에서 개발자 입맛에 맞는 양질의 포지션을 먼저 제안하고 채용 지원을 편하게 할 수 있도록 개선했습니다.

  • 테마가 있는 포지션, 적극 채용 중인 회사 외 기타 기능 추가
  • 가독성 향상을 위해서 회사 페이지 디자인, 레이아웃 개선 및 수정
  • 포지션 지원 -> 이력서 선택 시 페이지 이동을 삭제하여 플로우 간소화
  • 지원자들의 채용 진행 상태, 사이닝 보너스 등의 정보를 직관적으로 표현
1. 테마가 있는 포지션 페이지 추가

사용자들이 관심 있어 하는 테마를 강조하기 위해 새로운 채용 공고 리스트 페이지를 개발했습니다. 기존 페이지의 상단 영역은 테마와 관련된 이미지와 텍스트로 수정하여 사용자가 페이지에 접속했을 때 해당 테마를 즉시 인지할 수 있도록 했습니다. 또한, 포지션 카드의 디자인도 기존의 로고와 텍스트를 강조한 형태에서 이미지를 강조한 카드 디자인으로 변경하여 기존 채용 공고 리스트 페이지와 구별할 수 있도록 했습니다.

1-1. 테마가 있는 포지션 주제 선정

사용자 설문조사와 VOC를 통해 수집한 데이터를 활용하여 테마를 선정하고, 이를 기반으로 채용 서비스의 트래픽이 가장 높은 커리어 메인 페이지에 해당 테마를 노출시켰습니다.

1-2. 북마크 기능 추가

사용자들로부터 꾸준한 요청을 받았던 북마크 기능 북마크 리스트 조회 페이지를 개발했습니다. 북마크 기능을 추가함으로써, 채용 프로그램이 아닌 일반 채용 공고에 대한 지원율 상승이 유의미하게 나타났음을 확인할 수 있었습니다.

2. 회사 페이지 수정

이로 인해 지원자들은 이제 불필요한 페이지 이동 없이 직접 이력서를 선택하고 즉시 지원을 완료할 수 있게 되었습니다. 이는 결과적으로 지원 과정에서 발생하던 불필요한 단계를 줄이고, 지원자들은 채용 공고를 계속 읽으며 지원 여부를 판단할 수 있게 되었습니다.

4. 지원한 포지션 페이지 개선

취업 활동 증명서는 채용 서비스에서 사이닝 보너스와 함께 많이 발생하는 VOC 중 하나였습니다. 그러나 팀 내 우선순위에 밀려서 지원되지 않던 기능이었습니다. 이로 인해 CS팀은 요청이 들어올 때마다 로그를 확인하고 문서를 작성하여 전달해야 하는 번거로움을 겪었습니다.

그런 상황에서 지원한 포지션 개선 작업을 진행하게 되었는데, 이 때 취업활동 증명서 기능도 함께 개발하자고 의견을 제시해서 함께 진행하게 되었습니다.

이 기능은 채용 서비스를 이용하는 다수의 사용자를 위한 것은 아니었지만, 취업 활동 증명서를 필요로 하는 소수의 사용자와 CS팀원들에게는 꼭 필요한 기능이었습니다. 이로 인해 사용자들과 팀원들의 만족도가 크게 향상되어 저 또한 보람을 느낄 수 있었습니다.

코드 첨부

프로그래머스 이력서의 가장 큰 장점은 이력서에 코드를 첨부해서 내 개발 역량을 어필 할 수 있다는 점 입니다. 프로그래머스에서 코딩 테스트나 과제 테스트를 진행한 경우 별도의 정리 과정 없이 코드를 이력서에 첨부할 수 있습니다.

Problem

개발자들이 이력서를 만들기 위해 어떤 내용을 어떻게 정리해서 작성해야 할지 몰라 어려움을 겪고 있었습니다.

  • Github 링크를 올리면 채용 담당자들이 꼼꼼하게 읽는지 모르겠음
  • 개인적으로 진행한 사이드 프로젝트에 대해서 작성하고 싶은데 어떻게 해야할지 모르겠음
  • 온라인 저지 사이트에서 공부하면서 작성한 코드를 이력서에 활용 할 수 있으면 좋겠음
Solution

개발자들이 최소한의 노력으로 본인의 개발 역량을 드러낼 수 있도록 기능을 추가, 개선 했습니다.

  • 이력서 내에서 Github 계정을 연동하여 내가 작성한 코드 조회 기능 추가
  • 사이드 프로젝트 관련 Github 레포지토리를 이력서 내에 첨부 할 수 있도록 기능 추가
  • 프로그래머스 프로그램을 통해 작성한 코드를 이력서에 첨부 할 수 있도록 기능 추가
1-1. Github 계정 연동 기능 추가

이메일 주소 하단에 위치한 Github 계정 연동 버튼을 클릭하면, Github 계정에 Public으로 설정되어 있는 레포지토리를 이력서 페이지에 첨부할 수 있도록 디자인 했습니다.

1-2. Github 사용 언어

Github 전체 레포지토리를 종합하여 가장 많이 사용한 언어와 커밋 수를 이력서에서 간략하게 정리해서 표시했습니다.

1-3. Github 스마트 분석 코드

사이드 프로젝트 항목 저장소 링크에 github 레포지토리 주소를 입력하면 Github 스마트 분석 코드 보기버튼이 자동 생성됩니다.

1-4. Github 스마트 분석 코드

Github 스마트 분석 코드 보기 버튼을 클릭하면 해당 프로젝트 레포지토리에 푸시된 코드와 관련된 분석 자료를 페이지 이동 없이 바로 확인할 수 있게 디자인 했습니다.

이를 통해 개발자는 이력서에서 내 코드를 보여줄 수 있고, 채용 담당자는 이력서를 검토하면서 동시에 코드를 확인할 수 있어 적합한 지원자를 찾는데 도움이 되도록 디자인 했습니다.

2. 프로그램 코드 첨부
2-1. 프로그램 작성 코드 이력서 첨부

2-2. 프로그램 작성 코드 확인

이력서 기능

코드 첨부 기능 외에 이력서 본연의 역할을 하는데 꼭 필요한 기능을 추가, 개선 했습니다.

Problem

개발자 입장에서 채용 서비스가 너무 많아서 서비스 별로 이력서를 관리하는데 어려움을 겪고 있었습니다.

  • 이력서를 프로그래머스 밖에서도 사용할 수 있으면 좋겠다.
  • 내가 작성한 이력서에서 특정 항목만 비공개로 돌리고 싶다.
  • 프로필을 잘 잘성했는지 잘 모르겠다. 작성 가이드가 제공되면 좋겠다.
Solution

개발자들이 프로그래머스 이력서를 메인 이력서로 사용할 수 있게 필요한 기능을 정리해서 개발했습니다.

  • 이력서 외부 URL 공유 기능 추가 및 PDF 인쇄 기능 추가
  • 이력서 세부 항목 및 코딩 테스트 결과 공개 설정 기능 추가
  • 완성도 있는 이력서를 작성하는데 도움이 될 수 있는 완성도 기능 추가
1. 이력서 공유 및 URL 설정 기능

사용자가 공개 설정한 이력서를 프로그래머스 사이트 외부에서도 누구나 조회할 수 있도록 했습니다. 그리고 내가 원하는 URL까지 설정할 수 있는 기능을 추가 했습니다.

2. 이력서 PDF 저장 및 인쇄 기능

실제 채용 진행 과정에서 이력서를 PDF로 제출하거나 프린트해야 할 일이 많아 개발자와 기업 모두 프린트 기능 추가 요청이 많았습니다.

개발 과정에 있어서 PDF 변환에 Chrome 브라우저의 기능을 활용하기로 결정했습니다. 그래서 기존 이력서 화면에 적용된 디스플레이용 CSS 외에도 Print CSS 작업을 했습니다.

A4 프린트 화면에서 이력서 내용이 깨지지 않고 자연스럽게 출력되도록 레이아웃, 폰트, 사이즈, 컬러 등에 신경을 써서 디자인과 CSS 작업을 진행했습니다.

3. 세부 항목 공개 설정 기능

세부 항목 공개 기능이 없었을 때는 공개하고 싶지 않은 항목이 있으면 작성한 내용을 삭제해야만 했습니다. 이런 경험은 사용자 경험을 크게 저하시키기 때문에 개선 작업을 진행했습니다.

4. 이력서 프로필 가이드

개발자들의 이력서 작성률을 높이고 양질의 이력서를 보다 확보하기 위해, 상단 영역에 이력서 완성도 가이드 UI를 추가했습니다. 완성도 가이드를 도입함으로써, 이력서를 작성하는데 도움도 주고 메인 네비게이션과 실제 이력서 영역을 디자인적으로 확실하게 구분할 수 있게 되었습니다.

Business Design

비지니스 페이지는 채용 담당자들이 사용하는 어드민 페이지입니다. 채용 비지니스 페이지 중에서도 개발자 검색 리뉴얼 작업을 진행했습니다.

개발자 검색

개발자 검색 페이지는 채용 담당자들이 원하는 조건의 개발자를 직접 검색하고 채용 제안을 보낼 수 있는 페이지입니다. 채용 담당자들이 채용 성과를 높일 수 있도록 필요한 기능을 추가, 개선 했습니다.

Problem

채용 담당자들이 채용을 진행할때 회사에 직접 지원하는 개발자 외에 팀에 꼭 맞는 개발자를 찾는데 어려움을 겪고 있었습니다.

  • 채용에 실제 도움이 될만한 필터 조건이 더 추가되면 좋겠음
  • 누가 북마크 페이지에 개발자를 왜 추가 했는지 개발자에 대한 의견을 남길 수 있으면 좋겠음
  • 면접 제안 보낸 개발자들이 제안을 수락했는지 어떻게 처리했는지 트래킹 할 수 있으면 좋겠음
Solution

채용 담당자들이 팀에 꼭 맞는 좋은 개발자를 찾아서 면접 제안을 편하게 할 수 있도록 필요한 기능을 정리해서 추가, 개선했습니다.

  • 채용 담당자의 도움을 받아 채용에 필요한 조건을 추가
  • 팀내에서 커뮤니케이션 할 수 있도록 북마크 페이지에 추가된 개발자 그룹핑, 메모, 별점 기능 추가
  • 제안 관리 페이지를 새로 만들어서 개발자들이 제안을 어떻게 처리했고 어떤 단계인지 보여줌
1. 개발자 검색 페이지 레이아웃 변경

우선 가장 큰 변화는 개발자 리스트에서 타임라인을 제거한 것입니다. 개발자의 경력 사항을 직관적으로 보여주기 위해 타임라인 형태로 디자인했었으나, 각각의 개발자에게 다른 타임라인 형식이 적용되어 화면이 혼란스러워 보였다는 의견이 있었습니다. 또한, 채용 과정에 큰 영향을 미치지 않는다는 의견도 많아 과감하게 제거하였습니다.

또한, 채용 담당자들의 의견을 수렴하여 인재풀에 등록되어 있거나 지원 이력 또는 열람 내역이 있는 개발자는 목록에서 제외할 수 있게 필터를 추가 했고 우리 회사에 지원한 이력이 있는 개발자들은 언제 어떤 포지션에 지원했는지 확인할 수 있는 기능을 추가했습니다. 그리고 채용 담당자들이 요청한 이력서 업데이트 날짜 와 메모 유무를 표시해주었습니다.

이렇게 다양한 기능을 추가하고 디자인을 수정하면서 전체 리스트 형태를 카드 베이스로 변경하고 페이지 전체 폭 넓이를 조정하여 정보 스캔 시 눈에 피로감이 느껴지지 않도록 조정했습니다.

1-1. 포지션 카드 UI 수정

기존의 개발자 목록에서 타임라인, 아바타, 컬러 영역을 제거하고 카드 형태로 디자인을 수정하여 더 정리되고 깔끔한 모습으로 업데이트했습니다.

1-2. 제안 및 지원 이력 조회

이전에 제안을 거절한 개발자나 회사에 지원했지만 채용이 불발된 개발자들에게 중복 제안을 방지하기 위해 해당 기능을 추가했습니다.

1-3. 필터 기능 강화

채용 담당자들이 지원 이력와 열람한 후보 등을 필터를 통해 제외할 수 있는 기능을 요청해서 해당 기능을 서비스에 추가했습니다. 실제 고객의 목소리에 귀 기울여 필요한 기능을 추가하게 되었습니다.

2. 제안 관리 및 북마크
2. 제안 관리 페이지 리뉴얼
2-1. List -> Table 변경

리스트에서 테이블 형태로 디자인을 변경하여, 이전에 확인할 수 없었던 제안 포지션 제안 날짜 상태 제안 수락 단계에 따른 후보자 목록을 조회할 수 있도록 개선했습니다.

2-2. 북마크 관리 페이지 추가

북마크 관리 페이지가 추가되기 이전에는 제안/요청 페이지에서 후보자를 추가한 사실만 알 수 있었고, 채용에 필요한 정보 확인이나 후보자 그룹핑 같은 기능이 부재했습니다.

그래서 이번에 페이지를 추가 하면서 실제 채용 과정에 도움이 될 수 있도록 필요한 기능을 추가했습니다.

2-3. 평점 및 메모 기능 추가

후보자에 대한 평점 및 메모를 남길 수 있는 기능을 추가하여, 채용 담당자들이 후보자에 대한 코멘트를 남길 수 있게 되었습니다.
