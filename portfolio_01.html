프로그래머스의 프로덕트 디자이너로 재직하면서 개발자 전문 매칭 채용 서비스의 데스크탑, 모바일 디자인을 담당했습니다.

개발자들이 프로그래머스 플랫폼에서 원하는 회사에 취업할 수 있도록 기술 중심의 개발자 서비스를 만들기 위해 노력했습니다.

Service

많은 기업들이 능력있는 개발자를 적극적으로 찾고 있었지만, 기존 채용서비스의 이력서 제출, 1차 면접, 2차 면접, 합격 여부로 진행되는 일반적인 프로세스로는 기업이 필요로 하는 개발자를 찾는데 한계가 있었습니다. 이런 문제를 해결하고자 프로그래머스에서 역량있는 개발자들을 모아서 좋은 기업들과 연결될 수 있도록 개발자 채용에 특화된 플랫폼을 출시하게 되었습니다.

저는 개발 및 운영팀과 협력하여 채용 프로그램, 채용 공고, 개발자 특화 이력서부터 비지니스 개발자 검색 페이지에 이르기까지 핵심 경험을 전면적으로 개선했습니다.

Research

주요 지표(프로필 작성률, 채용 지원률, 채용 합격률 등), VOC, 설문조사, 인터뷰를 통해 얻은 피드백을 기반으로 필요한 기능을 정리하고, 해당 기능이 개발자 채용에 어느 정도 도움이 될지 살펴보았습니다. 그 후에 우선 순위를 정하여 사용자의 니즈를 해결하고 개발자 채용이라는 목표를 달성하기 위해 노력했습니다.

Design System

Typography

Icon

Colors

Career Design

프로그래머스 채용서비스는 기술 중심의 이력서를 바탕으로 개발자 평가를 통해 취업활동을 진행할 수 있는 국내 유일 개발자 전문 매칭 서비스입니다.

1. 메인 페이지 2. 채용 프로그램 3. 채용 공고

커리어 메인페이지는 채용 서비스와 관련된 모든 컨텐츠와 기능이 담겨있는 시작점이자 가장 높은 트래픽을 자랑하는 페이지입니다.

서비스 핵심 페이지이기 때문에 팀원들과 함께 개발자들이 성공적인 구직활동을 할 수 있도록 꾸준하게 관심과 애정을 가지고 서비스를 개선했습니다.

채용 프로그램

프로그래머스 채용 프로그램은 코딩 테스트를 한 번만 보면 이력서 한 장으로 다양한 기업에 동시 지원이 가능해서 구직 활동을 하는 개발자들의 참여율이 굉장히 높은 프로그램입니다.

Problem

프로그램에 참여하는 개발자 입장에서 몇가지 어려움이 있어서 지원 마감일까지 불필요한 눈치게임을 해야만 했습니다.

  • 지원 완료 이후 선택한 회사를 바꿀 수 없어서 선택이 어려움
  • 과거 지원 히스토리를 보고 싶은데 볼 수 없음
  • 테스트를 다시보면 이전 점수보다 오히려 떨어질까봐 걱정
  • 지원하고 싶은 회사에 다른 지원자들이 얼마나 지원 했는지 궁금함
Solution

팀원들과 함께 개발자 채용 과정에서 개발자들이 겪는 불필요한 부담을 줄여주기 위해 아래와 같이 서비스를 개선했습니다.

  • 지원 완료 이후 내가 선택한 회사를 변경 할 수 있는 기능 추가
  • 과거 지원 히스토리, 결과, 상태를 볼 수 있는 페이지 생성
  • 개발 실력이 검증된 상위 개발자 대상으로 테스트 과정 면제
  • 회사 로고, 지원율 정보를 추가해서 관련 정보 파악에 도움 되도록 개선
1. 선택한 회사 변경 기능

프로그램 지원 완료 이후에도 내가 선택한 회사 를 변경할 수 있도록 기능을 추가했습니다. 기존에는 이 기능이 없어서 수정하고 싶으면 지원을 취소하고 처음부터 다시 해야 하는 불편함이 있었습니다.

2. 프로그램 지원 내역

서비스 초기에는 채용 프로그램에 참여한 개발자 입장에서 지원 내역, 진행 상태등을 조회 할 수 있는 기능이 없었습니다. 그래서 이전 지원 내역을 따로 알려달라는 요청과 함께 관련 기능 개발을 해달라는 VOC가 많이 들어왔습니다.

사용자들의 피드백을 적극 반영하여 해당 기능을 개발하였고 그 결과 운영팀으로 들어오는 관련 VOC도 크게 줄고 사용자 만족도 역시 크게 상승했음을 확인할 수 있었습니다.

3. 프리패스 기능

양질의 지원자 참여율을 높이기 위해 이전 프로그램 테스트에 참여해서 개발 실력이 검증된 상위 개발자를 대상으로 테스트 과정을 면제해주는 프리패스 기능을 개발했습니다.

4. 프로그램 정보 페이지 개선

서비스 초기에 프로그램 참여 기업수가 많지 않을때는 채용 공고 목록 페이지를 텍스트로만 표시해도 사용자 입장에서 읽는데 무리가 없었는데 참여 기업이 40개 이상 늘어나게 되니까 텍스트로 구성된 테이블 정보가 잘 읽히지 않는 문제가 있었습니다. 그래서 로고를 추가해서 테이블 정보를 정리하고 개발자들의 회사 선택에 도움이 되도록 지원율을 표시해서 페이지 디자인을 개선했습니다.

채용 공고

채용 공고 페이지는 지원자들이 해당 포지션에 지원하기 위해 필수로 거쳐야하는 채용 프로세스의 시작점입니다. 이 페이지는 지원자의 입사 결정에 큰 영향을 미치고, 텍스트 정보가 많은 중요한 페이지이기 때문에 지원자들이 빠르게 정보를 스캔하고 쉽게 이해할 수 있도록 디자인을 구성했습니다.

Problem

개발자들이 채용 포지션 목록과 공고 페이지만 보고 자기한테 맞는 포지션을 찾는데 어려움을 겪고 있었습니다.

  • 내가 관심있을만한 회사를 추천받거나 모아서 보고 싶음
  • 회사 페이지 마다 정보의 양이 너무 달라서 어떤 회사는 읽기 어려움
  • 채용 지원을 하려고 이력서를 선택하고 지원하는 과정이 심리적으로 부담스러움
  • 내가 지원한 포지션 관리 페이지가 너무 불친절함 뭘 어떻게 해야될지 모르겠음.
Solution

서비스 내에서 개발자 입맛에 맞는 양질의 포지션을 먼저 제안하고 채용 지원을 편하게 할 수 있도록 개선했습니다.

  • 테마가 있는 포지션, 적극 채용 중인 회사 외 기타 기능 추가
  • 가독성 향상을 위해서 회사 페이지 디자인, 레이아웃 개선 및 수정
  • 포지션 지원 -> 이력서 선택 시 페이지 이동을 삭제하여 플로우 간소화
  • 지원자들의 채용 진행 상태, 사이닝 보너스 등의 정보를 직관적으로 표현
1. 테마가 있는 포지션 페이지 추가

사용자가 관심있어하는 테마를 강조하기 위해 새로운 채용 공고 리스트 페이지를 제작했습니다. 기존 페이지 상단 광고 영역은 테마와 관련된 이미지와 텍스트로 수정해서 사용자가 페이지에 들어왔을때 어떤 테마인지 인지할 수 있도록 했습니다. 그리고 포지션 카드도 기존 로고 + 텍스트를 강조한 형태에서 사용자들에게 친숙한 이미지를 강조한 카드 디자인으로 변경하여 기존 채용공고 리스트 페이지와 차별화를 두었습니다.

1-1. 테마가 있는 포지션 주제 선정

사용자 설문조사, VOC를 통해 얻은 데이터를 바탕으로 테마를 선정하여 채용 서비스 트래픽이 가장 높은 커리어 메인 페이지에 노출시켰습니다.

1-2. 북마크 기능 추가

VOC를 통해 꾸준히 요청이 들어왔었던 북마크 기능과 북마크 리스트 조회페이지를 만들었습니다. 북마크 기능을 추가하면서 채용 프로그램이 아닌 일반 채용 공고 대상 지원율 상승이 유의미하게 일어났음을 확인할 수 있었습니다.

2. 회사 소개 페이지 리뉴얼
2. 회사 페이지 수정

설문조사를 통해 회사 소개 페이지에 텍스트가 너무 많아 화면이 복잡해보인다는 피드백을 받았습니다. 그래서 전체 레이아웃을 카드 베이스로 수정해서 왼쪽 칼럼에는 텍스트 위주의 회사의 기본 정보를 배치했고 오른쪽 칼럼에는 사용자가 흥미를 느낄만한 연봉 사원수 그래프 지도 등의 부가 정보를 배치했습니다. 지원자들이 페이지에 들어왔을때 텍스트 때문에 부담을 느끼지 않도록 의도하여 디자인을 리뉴얼했습니다.

3. 채용 공고 페이지 수정

리뉴얼된 회사 소개 페이지에 맞춰서 일부 디자인 수정 작업을 진행했습니다. 그 후에 오른쪽 사이드 영역에 있던 기본 회사 정보는 채용 공고에 작성된 정보와 중복되는 것도 많았습니다. 그래서 삭제한 공간을 활용하여 내가 지원할 이력서를 선택할 수 있게 디자인을 변경하였습니다.

덕분에 지원자들은 이제 불필요한 페이지 이동없이 이력서를 선택하고 바로 지원 완료까지 할 수 있게 되었습니다. 결과적으로 지원 완료까지 소요되던 불필요한 단계를 줄이고 지원자 입장에서도 채용공고를 계속 읽으면서 지원 여부를 판단할 수 있어서 더 좋은 방향으로 개선했습니다.

4-2. 취업 활동 증명서 기능 추가

취업활동 증명서는 사이닝 보너스와 함께 채용 서비스 주요 VOC 중 하나였는데 그동안 팀내 우선순위에 밀려서 지원하고 있지 않던 기능이었습니다. 그래서 CS팀에서 요청이 들어올때 마다 일일이 로그를 뒤져가며 문서를 작성해서 전달해줬어야 했습니다.

그러던 와중에 지원한 포지션 개선 기능을 개발하기로 했고 취업활동 증명서랑 기능이 거의 비슷한거 같은데 이번 기회에 같이 지원해주는게 어떻겠냐라고 의견을 제시해서 함께 진행 할 수 있었습니다.

이 기능은 채용 서비스를 이용하는 대다수의 사용자를 위한 기능은 아니었지만 그래도 이 기능을 꼭 필요로 하는 소수의 사용자들과 CS팀원들의 만족도가 엄청 높아서 저도 많은 보람을 느낄 수 있었습니다.

4-3. 라벨, 텍스트 정리

그동안 포지션 타이틀 옆에 알록달록 보기 싫게 붙어있던 지원 상태, 사이닝 보너스 등의 라벨을 전부 제거하고 재배치했습니다. 그리고 사용자들이 쉽게 사용할 수 있도록 아이콘을 텍스트가 포함된 버튼 형태로 수정했습니다.

Profile / Resume

프로그래머스 이력서는 스택과 기술 중심의 이력서로 모든 채용 서비스를 이용하기 위해 필수로 작성되어야 하는 항목으로 많은 관심이 필요한 페이지입니다.

1. 이력서 2. 코드 첨부 3. 이력서 기능

개발자들이 자유롭게 이력서를 등록하고, 코딩테스트와 개발 과제에 도전하여 본인의 역량을 프로필에 어필할 수 있습니다.

학력과 경력만으로 나타내기 어려운 실력을 정확하게 보여주기 때문에 나의 개발 실력을 더 잘 나타낼 수 있습니다.

코드 첨부

다른 채용 서비스 이력서와 프로그래머스 이력서를 비교했을때 프로그래머스의 가장 큰 장점은 이력서에 코드를 첨부해서 내 개발 역량을 어필 할 수 있다는 점 입니다. 그래서 개인 프로젝트나 프로그래머스에서 코딩 테스트나 과제 테스트를 진행한 경우 별도의 정리 과정없이 코드를 이력서에 첨부하면 바로 이직/구직 활동을 진행할 수 있습니다.

Problem

개발자들이 이력서를 만들기 위해 어떤 내용을 어떻게 정리해서 작성해야 할지 몰라 어려움을 겪고 있었습니다.

  • Github 링크를 올리면 채용 담당자들이 꼼꼼하게 읽는지 모르겠음
  • 개인적으로 진행한 사이드 프로젝트에 대해서 작성하고 싶은데 어떻게 해야할지 모르겠음
  • 온라인 저지 사이트에서 공부하면서 작성한 코드를 이력서에 활용 할 수 있으면 좋겠음
Solution

개발자들이 최소한의 노력으로 본인의 개발 역량을 드러낼 수 있도록 기능을 추가/개선 했습니다.

  • Github 계정을 연동해서 내가 커밋한 코드를 이력서 내에서 보여줌
  • 사이드 프로젝트 관련 Github 레포지토리를 이력서 내에 첨부 할 수 있도록 기능 추가
  • 프로그래머스 프로그램을 통해 작성한 코드를 이력서에 첨부 할 수 있도록 기능 추가
1-1. Github 계정 연동 기능 추가

이메일 주소 아래에 Github 계정 연동 버튼을 누르면 Github 계정 내 Public 공개 되어 있는 데이터를 이력서 페이지에 첨부할 수 있도록 디자인 했습니다.

1-2. Github 사용 언어

내 Github 전체 레포지토리를 통틀어서 어떤 언어를 가장 많이 사용했는지, 커밋은 얼마나 했는지 이력서 내에서 간략하게 요약, 정리해서 표시했습니다.

1-3. Github 스마트 분석 코드

사이드 프로젝트 항목 저장소 링크에 github 레포지토리 주소를 입력하면 Github 스마트 분석 코드 보기버튼이 자동 생성됩니다.

1-4. Github 스마트 분석 코드

이력서, 포트폴리오를 작성할때 개인 사이드 프로젝트를 어떻게 설명해야할지 모르는 개발자들이 많았습니다. 그래서 개발자들이 크게 고민하지 않고 키워드 위주로 내용을 입력하면 완성도 있는 항목이 작성될 수 있게 했습니다. 그리고 중요한 내용은 코드 자체로 보여줄 수 있게끔 Github 스마트 분석 코드 보기버튼을 누르면 해당 프로젝트 레포지토리에 푸시한 코드와 관련된 분석 자료를 페이지 이동 없이 바로 볼 수 있게 디자인했습니다.

항목을 입력하는 개발자도 쉽게 코드 내용을 보여줄 수 있고 채용 담당자 입장에서도 이력서를 검토하면서 동시에 코드도 함께 확인할 수 있기 때문에 적합한 지원자를 찾는데 큰 도움이 된다는 피드백을 받았습니다.

2. 프로그램 코드 첨부
2-1. 프로그램 작성 코드 이력서 첨부

개발자들이 프로그래머스 사이트 안에서 가장 많이 이용하는 서비스인 코딩 테스트(스킬 체크)와 채용 프로그램에 지원해서 응시하는 1차 코딩 테스트와 2차 과제 테스트 코드를 이력서에 첨부 할 수 있게 디자인 했습니다.

테스트 상세 결과 보기, 코드 보기, 스킬 체크 뱃지 아이콘을 누르면 각 항목에 해당하는 코드를 볼 수 있습니다.

2-2. 프로그램 작성 코드 확인

개발자가 작성한 코드를 이력서에서 바로 확인할 수 있게 디자인하여 개발자들의 이력서를 더 풍성하게 보여줄 수 있도록 디자인했습니다.

이력서 기능

코드 첨부 기능 외에 이력서 본연의 역할을 하는데 꼭 필요한 기능을 추가/개선 했습니다.

Problem

개발자 입장에서 채용 서비스가 너무 많아서 서비스 별로 이력서를 관리하는데 어려움을 겪고 있었습니다.

  • 이력서 외부 URL 공유 기능 추가 및 PDF 인쇄 기능 추가
  • 이력서 세부 항목 및 코딩 테스트 결과 공개 설정 기능 추가
  • 완성도 있는 이력서를 작성하는데 도움이 될 수 있는 완성도 기능 추가
1. 이력서 공유 및 URL 설정 기능

사용자가 공개 설정한 이력서를 프로그래머스 사이트 밖에서도 누구나 조회할 수 있도록 하고 내가 원하는 URL까지 설정할 수 있는 기능을 디자인 했습니다.

2. 이력서 PDF 저장 및 인쇄 기능

실제 채용 진행 과정에서 이력서를 PDF로 제출 해야 하거나 프린트 해야 할 일이 많다 보니 개발자측에서도 그렇고 채용을 진행해야 하는 기업측에서도 프린트 기능을 추가해달라는 VOC가 많았습니다.

개발 과정에 있어서 Chrome 브라우저 PDF 변환 기능을 사용하기로 했는데 그렇게 하기 위해서 기존 이력서 화면에 적용되어있던 디스플레이용 CSS외에 추가로 Print CSS 작업을 진행해야했습니다.

디스플레이 화면의 이력서 내용을 A4 프린트 화면에서 깨지지 않고 자연스럽게 출력하기 위해서 레이아웃, 폰트, 사이즈, 컬러 등을 신경써서 디자인 및 CSS 작업을 진행했습니다.

3. 세부 항목 공개 설정 기능

개발자 입장에서 세부 항목 공개 기능이 없었을때는 공개하고 싶지 않은 항목이 있으면 기껏 작성해놓은 내용을 삭제해야만 했습니다. 팀내에서 논의했을때 이런 경험이 사용자 경험을 크게 해친다고 판단하여 빠르게 개선 작업을 진행하였습니다.

4. 이력서 프로필 가이드

개발자들의 이력서 작성률을 올려서 양질의 이력서를 확보하기 위해서 상단 영역에 이력서 완성도 가이드 UI를 추가했습니다. 완성도 가이드를 추가하면서 평소에 문제라고 생각했었던 메인 네비게이션 이력서 헤더(내 프로필) 실제 이력서 내용 영역을 디자인적으로 명확하게 구분하자고 제안을 했고 팀원들도 함께 동의 얻어서 디테일한 디자인 부분도 함께 수정할 수 있었습니다.

Business Design

비지니스 페이지는 일반 개발자들이 아닌 채용을 진행하는 채용 담당자들이 사용하는 어드민 페이지입니다. 채용 비지니스 페이지 중에서도 개발자 검색 리뉴얼 작업을 진행했습니다.

개발자 검색

개발자 검색 페이지는 채용 담당자들이 원하는 조건의 개발자를 직접 검색하고 채용 제안을 보낼 수 있는 페이지입니다. 이 페이지를 통해서 채용 담당자들이 채용 성과를 높일 수 있도록 채용 및 제안에 필요한 기능을 추가, 개선 했습니다.

Problem

채용 담당자들이 채용을 진행할때 회사에 직접 지원하는 개발자 외에 팀에 꼭 맞는 개발자를 찾는데 어려움을 겪고 있었습니다.

  • 채용에 실제 도움이 될만한 필터 조건이 더 추가되면 좋겠다.
  • 누가 북마크 페이지에 개발자를 왜 추가 했는지 알 수 있었으면 좋겠다. 개발자에 대한 의견을 남길 수 있으면 좋겠다.
  • 면접 제안 보낸 개발자들이 제안을 수락했는지 어떻게 처리했는지 트래킹 할 수 있으면 좋겠다.
Solution

채용 담당자들이 팀에 꼭 맞는 좋은 개발자를 찾아서 면접을 제안을 편하게 할 수 있도록 필요한 기능을 정리해서 추가, 개선했습니다.

  • 채용 담당자의 도움을 받아 채용에 필요한 조건을 추가
  • 팀내에서 커뮤니케이션 할 수 있도록 북마크 페이지에 추가된 개발자 그룹핑, 메모, 별점 기능 추가
  • 제안 관리 페이지를 새로 만들어서 개발자들이 제안을 어떻게 처리했고 어떤 단계인지 보여줌
1. 개발자 검색 페이지 레이아웃 변경

우선 가장 큰 변화는 개발자 리스트에 포함되어 있던 타임라인을 제거했습니다. 개발자의 학력경력 사항을 직관적으로 보여주기 위해 타임라인 형태로 디자인 했었는데 개발자 별로 표시되는 타임라인 형태가 너무 달라서 화면이 어지러워 보인다는 의견도 있었고 결정적으로 채용 과정에 있어서 큰 영향을 끼치지 않는다는 의견이 많아서 과감하게 걷어냈습니다.

추가로 채용 담당자들의 의견을 수렴하여 인재풀에 등록 되어 있거나 지원 이력 혹은 열람 내역이 있는 개발자는 목록에서 제외할 수 있게 필터 조건을 추가 했습니다. 그리고 실제 우리 회사에 지원한 이력이 있는 개발자들은 언제 어떤 포지션에 지원했는지 확인할 수 있는 기능을 추가했습니다. 그리고 채용 담당자들이 강력하게 요구했었던 이력서 업데이트 날짜 및 메모 존재 유무를 표시해주었습니다.

이렇게 여러 기능을 추가하고 디자인을 수정하면서 전체 리스트 형태도 카드 베이스로 수정하고 페이지 전체 폭 넓이도 기존에는 타임라인을 잘 보여주기 위해서 브라우저 full-width 넓이로 설정을 했었는데 타임라인을 걷어내면서 채용 담당자들이 정보를 스캔 하는데 눈에 피로감이 들지 않도록 적정 넓이로 조정해주었습니다.

1-1. 포지션 카드 UI 수정

기존 개발자 목록에서 타임라인, 아바타, 컬러 영역을 삭제하고 카드 형태로 디자인을 수정해서 기존 보다 더 정리되어 보이도록 수정했습니다.

1-2. 제안 및 지원 이력 조회

이전에 제안을 보냈는데 거절했거나 회사에 지원한적이 있는데 채용이 불발된 개발자들에게 다시 제안하는 일이 없도록 해당 기능을 추가했습니다.

1-3. 필터 기능 강화

채용 담당자들이 이력서 최신 업데이트 날짜와 채용 지원 이력기능이 있었으면 좋겠다고 의견을 줘서 서비스에 반영했습니다. 팀내에서 이야기 할때는 그렇게 중요한 기능인지 모르겠다는 의견이 많았었는데 실제 고객의 입장에서는 꼭 필요하다고 해서 팀내에서 논의 후 반영하게 되었습니다.

2. 제안 관리 및 북마크
2. 제안 관리 페이지 리뉴얼
2-1. List -> Table 변경

리스트에서 테이블로 디자인을 바꾸면서 이전에는 확인 할 수 없었던 제안 포지션제안 날짜상태제안 수락 단계에 따른 후보자 목록을 조회 할 수 있게 개선했습니다.

2-2. 북마크 관리 페이지 추가

북마크 관리 페이지가 추가되기전에는 제안 관리 페이지와 마찬가지로 어떤 후보자를 추가 했는지만 알 수 있었고 채용에 필요한 정보 확인이나 후보자 그룹핑 같은 기능이 없었습니다.

그래서 이번에 페이지를 추가, 리뉴얼 하면서 실제 채용까지 이루어질 수 있도록 꼭 필요한 기능을 추가했습니다.

이 작업을 통해 단순 디자인 작업 외에도 팀내에서 처음 MVP를 설정할때 사용자 입장에서 필요한 기능을 많이 어필해야겠다는 생각을 많이 했습니다. 그래서 이전보다 팀 내에서 의견 개진도 많이 하고 팀이 좋은 방향으로 나아가도록 설득하기 위해 근거를 마련하려고 공부도 많이했었던 기억이 납니다. 처음으로 중간 커뮤니케이터로써 제 역할에 대해서 다시 생각해볼 수 있었던 귀중한 프로젝트였습니다.

2-3. 평점 및 메모 기능 추가

후보자에 대해서 평점 및 메모를 남길 수 있는 기능을 추가했습니다. 메모를 남겨 후보자에 대해 코멘트를 남길 수 있게 되어 채용 담당자들의 긍정적인 피드백을 받을 수 있었습니다.
