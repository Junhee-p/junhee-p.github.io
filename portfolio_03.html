프로그래머스에서 프로덕트 디자이너로 일하면서 온라인 시험 감독 서비스 모니토를 성공적으로 출시했습니다.

기업들이 온라인 비대면 환경에서도 개발자의 역량을 정확하게 평가할 수 있도록 디자인 했습니다.

Timeframe
2020.02 - 2020.04
Role
UI/UX
Platforms
Desktop, Mobile
Category
Coding, Test

Service

코로나19의 발생으로 인해 기존의 대면 채용 프로세스를 더 이상 유지할 수 없게 되었습니다. 특히, 개발자 코딩 테스트는 공정성을 위해 응시자를 특정 장소에 모아 감독하며 진행하는 것이 일반적이었으나, 이러한 방식은 더 이상 실시할 수 없었습니다.

우리는 시장의 요구에 신속하게 대응하기 위해 프로그래머스의 디자인 시스템을 활용하여 단기간 내에 서비스 디자인을 진행했습니다. 이를 위해 직접 코드 작업을 수행하고 Git을 통해 개발자들과 협업하여 성공적으로 서비스를 출시할 수 있었습니다.

Test

테스트 시작 전 데스크탑, 노트북 및 모바일 기기 카메라를 이용한 신분증 인증 및 화상 설정 프로세스를 디자인 했습니다.

1. 테스트 안내 2. 사전 동의 3. 신분증 인증 4. 화상 기기 설정

테스트 시작 전에 테스트 응시자가 꼭 알고 있어야할 내용을 안내해주는 페이지입니다.

  • 테스트 응시자 확인
  • 테스트 기본 정보

원활한 테스트 진행을 위해 응시자의 동의가 필요한 부분이 있어, 응시자들이 내용을 확인하고 동의할 수 있도록 테스트 안내 후에 사전 동의 단계를 추가했습니다.

본인 인증을 위해 테스트 응시자가 모바일 접속 링크로 이동하여 인증 코드를 입력하고, 직접 신분증을 촬영하여 업로드할 수 있도록 디자인했습니다.

Step 1. 접속 링크 페이지 이동

모바일 기기로 웹 페이지에 기재되어 있는 접속 링크로 이동합니다.

Step 2. 인증 코드 입력

웹 페이지에 기재되어 있는 인증 코드를 모바일 브라우저에 작성합니다.

Step 1, 2. 접속 링크 이동 및 인증 코드 입력

웹 페이지에 기재되어 있는 접속 링크로 이동해서 인증 코드를 입력합니다.

Step 3. 촬영 안내 페이지 이동

안내 사항을 읽고 신분증이 잘 보이도록 핸드폰 카메라로 촬영합니다.

Step 4. 신분증 촬영 및 제출

모바일 기기로 신분증을 촬영 한 후에 업로드합니다.

Step 5. 화상 기기 설정

다시 웹 페이지로 돌아와서 2. 핸드폰으로 접속 옆에 뱃지가 신분증 제출 완료로 바뀐 걸 확인하고 우측 상단에 화상 기기 설정 버튼을 클릭합니다.

Step 1. 컴퓨터 화면 및 웹캠 공유

컴퓨터를 활용해서 내 컴퓨터 출력 화면과 웹캠 카메라 화면을 감독관에게 공유합니다.

Step 2. 카메라 연결 안내 페이지 이동

안내 사항을 읽고 카메라를 연결합니다.

Step 3. 카메라 연결 완료

카메라가 내 주변 상태를 잘 찍어주는지 확인하고 웹페이지로 돌아가 감독관 화면을 체크합니다.

Step 4. 카메라 연결 확인

감독관에게 재생되는 핸드폰 화면에 내 모습이 잘 나오는지 확인하면 기존 프로그래머스 테스트 단계로 넘어갑니다.

1. 모니터 화면 공유 버튼 클릭: 응시자는 자신의 컴퓨터 화면을 공유합니다.

2. 카메라/마이크 공유 버튼 클릭: 응시자는 자신의 정면 얼굴 화면과 음성을 전송합니다.

3. 주변 상태 비디오 촬영 : 응시자는 주변 상태를 실시간으로 촬영하여 감독관과 공유합니다.

감독관은 실시간으로 응시자의 컴퓨터 화면, 정면 얼굴 화면, 그리고 주변 상태를 확인하여 부정행위를 예방할 수 있습니다.

Manager

테스트 응시자들을 관리 감독하는 페이지를 디자인 했습니다. 서비스 초기에는 그리드 뷰와 와이드 뷰 라는 개념으로 페이지를 나눴는데 그리드 뷰 작업을 진행했습니다.

Sidebar

사이드바를 통해 테스트 응시자의 화상 기기 연결 상태를 확인할 수 있습니다. 또한, 실제 화상 감독 연결 여부와 상관없이 원하는 응시자만 선택하여 자세히 살펴볼 수 있습니다.

Error

화상 기기가 연결되지 않은 경우, 왜 연결되지 않았는지 자동으로 알려주도록 디자인 했습니다.

Design System

해당 시기에는 CSS를 사용하여 디자인 컴포넌트를 직접 수정하고 Git(Sourcetree)을 통해 개발자들과 직접 협업했었습니다. 그래서 촉박한 일정에서도 미리 정의한 컴포넌트를 활용하여 웹 브라우저에서 바로 디자인을 구현할 수 있었습니다.

지금 돌이켜보면, 일상적으로 디자인 시스템 작업을 꾸준히 수행해온 덕분에 갑작스러운 서비스 출시 과정에서도 신속하게 대응할 수 있었던 것 같습니다.
