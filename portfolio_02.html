2019년 부터 2022년 까지 개발자 설문조사 디자인과 코드 작업을 진행했습니다.

매년 진행할때마다 주어진 시간안에 디자인과 코드 작업을 함께 진행해야해서 어려운점이 많았습니다. 그래서 그런지 몰라도 항상 뿌듯하면서도 아쉬움이 남는 작업입니다.

Timeframe
2019 ~ 2022
Role
Design, Publishing
Platforms
Desktop, Mobile
Category
Survey Design

Overview

개발자들이 스택오버플로우, 해커랭크에서 만든 개발자 설문조사 결과를 재밌게 보고 공유하는걸 보면서 팀내에서 우리도 한국형 개발자 설문조사 컨텐츠를 만들어보는게 어떻겠냐는 이야기가 나오게 되었습니다. 그래서 자연스럽게 한국 개발자에 초점을 맞춰 컨텐츠를 제작하게 되었습니다.

Colors

많은 양의 데이터를 다루는 컨텐츠이기 때문에 사람들이 흥미를 잃지 않고 정보를 받아들이고 이해하는데 도움이 되도록 컨텐츠 레이아웃 부터 차트 디자인까지 컬러 디테일에 신경을 많이 썼습니다.

1. Light Theme

다크 테마로 디자인된 이전 설문조사 컨텐츠는 단조로워 보이는 단점이 있었습니다. 그래서 전체 컬러 테마를 라이트 테마로 변경하여 컬러를 전보다 과감하게 사용했습니다. 사용자에게 밝고 긍정적인 느낌을 전달하여 컨텐츠를 끝까지 읽을수 있도록 디자인했습니다.

2. Color Selection

디자이너의 눈대중이 아닌 정확한 기준을 가지고 색조, 채도, 명도 값을 조정해서 컬러를 선택했습니다.

3. UI Color

Tab이랑 그래프에 똑같은 컬러를 사용해서 사용자가 탭을 눌렀을때 헷갈리지 않도록 디자인했습니다.

4. Color Palette

페이지 디자인에 잘 어울리는 차트 컬러 팔레트를 만들어서 그래프에 적용시켰습니다.

Chart Design

데이터 왜곡 없이 정확한 내용을 전달하기 위해 몇가지 원칙을 세워서 디자인했습니다.

1. 모든 차트 데이터의 시작점은 '0'

시작점을 어떻게 설정하는지에 따라 똑같은 데이터도 다르게 보일 수 있습니다. 그래서 데이터를 해석할때 왜곡이 일어나지 않도록 모든 그래프의 시작점을 '0'으로 설정했습니다.

2. Bar Chart 데이터 내림차순 정렬

차트 데이터를 내림차순으로 정렬해서 그래프를 보는 사람이 데이터 해석을 쉽게 할 수 있도록 했습니다.

(하위 탭에 속한 그래프의 경우 대표 그래프와의 원활한 비교를 위해 정렬하지 않았습니다.)
3. Doughnut Chart의 최대 조각 갯수는 4개

도넛 차트의 도넛 조각이 5개가 넘어가면 그래프 해석이 어렵기 때문에 4개가 넘는 경우는 도넛 차트로 표현하지 않았습니다.

4. 지역 데이터는 차트 대신 지도로 표현

지역 데이터의 경우 차트 형태로 표현하면 정보 해석이 어렵습니다. 그래서 직관적으로 이해할 수 있도록 지도를 활용해서 표현했습니다.

5. Image, Icon 활용

후반부의 흥미성 내용은 텍스트 보다 이미지로 표현하는게 좋겠다고 생각했습니다. 그래서 사람들의 흥미를 이끌어낼 수 있도록 아이콘을 적극적으로 사용했습니다.
